# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Document Upload & AI Extraction.
> Surfaces in scope: `/upload`, `/documents/[id]`, and home page (`/`) update with "Dokumente hochladen" CTA + "Zuletzt hochgeladen" list.
> Extends Phase 1's contract. Typography, spacing, accent, and destructive color are unchanged — one new warning color is introduced (reserved for confidence indicators only).
> Generated by gsd-ui-researcher (autonomous mode).

---

## Surfaces in Scope

| Surface | Route | Purpose |
|---------|-------|---------|
| Upload page | `/upload` | Dropzone + per-file batch status list |
| Document detail | `/documents/[id]` | Extracted fields table with confidence badges + PDF preview |
| Home update | `/` | Primary "Dokumente hochladen" CTA + "Zuletzt hochgeladen" (last 5 uploads) |
| Inline upload errors | `/upload` (per row) | Per-file error message + "Erneut versuchen" affordance |
| Toasts | Global (Sonner) | Batch-complete success, unexpected network/server errors |

Out of scope (deferred): inline edit of extracted fields (Phase 3), authority lookup display (Phase 3), case grouping (Phase 4), re-upload (Phase 5).

## Color

60/30/10 split unchanged from Phase 1. **One new state color added: warning (amber), reserved exclusively for medium-confidence indicators.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (light) | Page background, main surface |
| Secondary (30%) | `#F5F5F5` (neutral-100) | Card backgrounds (dropzone card, fields panel, recent-list card), table header row, header bar |
| Accent (10%) | `oklch(0.21 0.04 265)` (slate-900, ≈ `#0F172A`) | Primary CTAs ("Dokumente hochladen", "Dateien auswählen"), focus rings |
| Destructive | `oklch(0.58 0.22 27)` (red-600, ≈ `#DC2626`) | Low-confidence badge, "— nicht erkannt" value text, error status badge, inline error copy |
| **Warning (new)** | `oklch(0.77 0.15 80)` (amber, ≈ `#D97706` at text-safe contrast) | **Medium-confidence badge ONLY** |

### Warning color — justification and constraint

A distinct medium-confidence signal is required: conflating medium with low (destructive red) over-alarms the operator; conflating medium with high (neutral) hides review-worthy fields. The warning token is therefore introduced as a **state color, not a new accent**. It appears nowhere else in the UI.

- Implementation: add as a Tailwind v4 `@theme` token named `--color-warning` with a foreground pair `--color-warning-foreground` (near-black `oklch(0.2 0 0)` for accessible contrast on the amber fill).
- Contrast: fill + dark foreground tested at ≥ 4.5:1 for 14px label text.
- Tailwind v4 OKLCH is used throughout to match Phase 1.

### Accent — reserved for (unchanged + extended)

- Primary CTA buttons: "Dokumente hochladen" (home), "Dateien auswählen" (upload dropzone button), "Zur Überprüfung" (document detail — non-functional placeholder for Phase 3).
- Focus ring on all interactive elements (inherited from shadcn defaults).
- Active row highlight in tables (left border 2px accent when a row has keyboard focus).

**Accent is NOT used for:** status badges (queued/uploading/extracting/done), confidence badges, body text, section headers, or decorative elements.

### Destructive — reserved for (unchanged + extended)

- Inline form / upload error text.
- Error-state upload row badge ("Fehler").
- Low-confidence badge (label "Niedrig").
- "— nicht erkannt" extracted-value placeholder text.
- Sonner toast destructive icon.

### Warning — reserved for (new, strict)

- Medium-confidence badge (label "Mittel") on `/documents/[id]`.
- Nothing else. Not used for in-progress states, not used for borders, not used as a decorative tint.

### Status badge palette (neutral, non-accent)

Per-file upload status (`/upload` batch list) uses shadcn Badge variants, NOT accent/destructive/warning except where explicitly listed above:

| Status | Badge variant | Rationale |
|--------|---------------|-----------|
| In Warteschlange | `secondary` | Neutral waiting state |
| Wird hochgeladen | `secondary` + inline spinner | Neutral with progress affordance |
| Wird analysiert | `secondary` + inline spinner | Neutral with progress affordance |
| Fertig | `outline` | Non-shouting completion state |
| Fehler | `destructive` | Matches destructive palette rule |

Text-on-color and borders remain as declared in Phase 1.

## Interaction Contract

| Interaction | Expected Behaviour |
|-------------|--------------------|
| Drag file over dropzone | Border changes from `border-dashed` at `border` color to solid at accent color; primary copy swaps to `Zum Hochladen loslassen` |
| Drop valid files | Rows append to `Aktuelle Uploads` list in `In Warteschlange` state; each transitions to `Wird hochgeladen` → `Wird analysiert` → `Fertig` or `Fehler` |
| Drop rejected file (wrong type / size / batch cap) | Inline red message appears under dropzone for 5s; no row added |
| Click `Dateien auswählen` | Opens native OS file picker; same validation rules apply |
| Concurrent uploads | Up to 3 files analyzed in parallel (CONTEXT D-Discretion). Remaining files stay in `In Warteschlange` and visibly transition as slots free |
| Row in `Fertig` state | Entire row is clickable → navigates to `/documents/[id]`; keyboard: Enter/Space on focused row triggers same nav |
| Row in `Fehler` state | Click on `Erneut versuchen` re-runs extraction (Phase 2 scope: upload succeeds but extraction failed — file is already on disk); status returns to `Wird analysiert` |
| Browser refresh mid-batch | Active uploads are lost; completed uploads persist (they reached DB). User sees only the persisted ones in `Zuletzt hochgeladen` |
| Navigate away mid-batch | Sonner toast warns `Uploads laufen noch — Seite verlassen?` via `beforeunload` on rows still in `Wird hochgeladen` |
| `/documents/[id]` during extraction | Table renders with Skeleton rows (one per expected field); confidence badges absent; `Analyse läuft …` subtext |
| `/documents/[id]` after extraction | Table fully populated; missing values show `— nicht erkannt` in destructive color; each populated value has a confidence badge |
| PDF preview | Inline `<iframe>` of the stored PDF (CONTEXT D-Discretion, simplest option). Min-height 480px. Fallback: `Dokument kann nicht angezeigt werden. <a>Als PDF öffnen</a>` if iframe blocked |
| Keyboard navigation | Dropzone focusable (Tab), activates file picker on Enter/Space. Batch rows focusable in DOM order. Document detail: Tab order = PDF preview → each field row (for future edit focus) → `Zur Überprüfung` CTA |
| Focus visible | Accent focus ring on all interactive elements (inherited from shadcn) |

## Layout Contract

### Home (`/`) update

- Reuse Phase 1 header shell unchanged.
- Main content: centered container, `max-width: 960px`, horizontal auto margins, top offset `32px` (xl), vertical gap `48px` (2xl) between sections.
- Section 1: Heading `Übersicht` (24px/600) + primary CTA `Dokumente hochladen` aligned right of heading on ≥768px, stacked below on <768px. Gap `16px` (md).
- Section 2: Heading `Zuletzt hochgeladen` (24px/600) + shadcn Card wrapping a Table with columns `Dateiname | Hochgeladen | Status | ` (last column is row-action `Öffnen`). Max 5 rows, no pagination in Phase 2.
- Empty state: same Card, centered body content at `48px` vertical padding, primary CTA inside card.

### Upload (`/upload`)

- Reuse header shell.
- Main content: centered container, `max-width: 720px`, top offset `32px` (xl), vertical gap `24px` (lg) between heading block, dropzone card, batch list.
- Dropzone card: shadcn Card with `border-dashed` outer frame, min-height `192px`, internal padding `24px` (lg), centered column layout: icon (lucide `Upload`, 24px, muted) → primary copy → `oder` muted text → `Dateien auswählen` button.
- Batch list: Card, padding `16px` (md) horizontal × `8px` (sm) vertical; each row is a 3-column grid on ≥768px (filename · status badge + progress · actions), stacks on <768px. Row vertical padding `12px` (outside 4-grid → round to `16px` md).

### Document detail (`/documents/[id]`)

- Reuse header shell.
- Breadcrumb row (Body size, muted): `Übersicht / Dokument` with `/` as a static separator. Home link functional, current page not a link.
- Heading block: `Dokument` (24px/600) + filename + upload timestamp subtext.
- Two-column layout on ≥1024px: left PDF preview (flex 1), right extracted-fields panel (`min-width: 400px`, `max-width: 480px`). Gap `32px` (xl).
- Below 1024px: stack vertically — PDF preview first, fields table second. Gap `24px` (lg).
- PDF preview: shadcn Card with `0` internal padding, iframe fills card, min-height `480px`.
- Fields panel: shadcn Card with padding `24px` (lg). Table inside. Below table: row with `Zur Überprüfung` CTA (disabled, tooltip); aligned right.

### Component sizing

- Button heights (inherited from shadcn new-york): default 40px, sm 36px, ghost 36px.
- Input heights (inherited): default 40px.
- Badge heights (inherited): 20px.
- Table row height: 44px (md row, accommodates badge + comfortable tap/click).

## Confidence Badge Specification

Precise because this is the only net-new visual contract introduced in Phase 2.

| Level | Label (DE) | Badge variant | Fill color | Text color | Usage |
|-------|------------|---------------|------------|------------|-------|
| high | `Hoch` | `secondary` | `#F5F5F5` (neutral-100) | `#0A0A0A` | Default shadcn Badge secondary variant |
| medium | `Mittel` | `warning` (new) | `oklch(0.77 0.15 80)` | `oklch(0.2 0 0)` | New `--color-warning` / `--color-warning-foreground` tokens |
| low | `Niedrig` | `destructive` | `oklch(0.58 0.22 27)` | `#FFFFFF` | Reuse shadcn Badge destructive variant |

- Placement: right-most cell of each row in the extracted-fields table.
- Spacing: `4px` (xs) between badge text and any prefixed dot/icon (if used). No icon in Phase 2 — text-only badge.
- When value is null (`— nicht erkannt`): show destructive badge `Niedrig`. Do not omit the badge.
