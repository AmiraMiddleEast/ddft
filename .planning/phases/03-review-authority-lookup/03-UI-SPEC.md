# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Review & Authority Lookup.
> Surface in scope: `/documents/[id]/review` — two-column layout (PDF preview left, editable form right) with an authority result panel rendered below the form after a successful lookup.
> Extends Phase 1 + Phase 2 contracts. Typography, spacing, accent, destructive, and warning tokens are unchanged. No new color tokens are introduced.
> Generated by gsd-ui-researcher (autonomous mode).

---

## Surfaces in Scope

| Surface | Route | Purpose |
|---------|-------|---------|
| Review page | `/documents/[id]/review` | Operator verifies and edits the 6 extracted fields side-by-side with the PDF, then triggers authority lookup |
| Inline field validation errors | `/documents/[id]/review` (per field) | Zod validation failures surface under the field |
| Authority result panel | `/documents/[id]/review` (below form, rendered post-submit) | Displays the resolved Vorbeglaubigung authority with routing breadcrumb, contact block, special rules, and [PRÜFEN] warning |
| Authority not-found panel | `/documents/[id]/review` (below form) | Displays when the resolver returns `not_found` |
| Authority ambiguous panel | `/documents/[id]/review` (below form) | Displays when the resolver returns multiple candidates (`lookup_status='ambiguous'`) |
| Toasts | Global (Sonner) | Unexpected server errors; success toast on successful approval + lookup |

Out of scope (deferred): case grouping (Phase 4), Laufliste PDF generation (Phase 4), re-upload (Phase 5), Behörden admin CRUD (Phase 5).

## Spacing Scale

Unchanged from Phase 1/2. All values are multiples of 4.

| Token | Value | Usage in Phase 3 |
|-------|-------|------------------|
| xs | 4px | Icon–label gap inside badges; gap between confidence pill and field-edited dot indicator |
| sm | 8px | Vertical gap between field label and control; gap between address lines in authority block |
| md | 16px | Default gap between form fields; Card inner padding-block; gap between form CTAs |
| lg | 24px | Card inner padding (horizontal); section gap inside authority result panel; gap between left PDF column and right form column on <1024px |
| xl | 32px | Page top offset; column gap between PDF preview and form panel on ≥1024px; vertical gap between form card and authority result panel |
| 2xl | 48px | Not used in Phase 3 |
| 3xl | 64px | Not used in Phase 3 |

Exceptions:
- PDF preview iframe: `min-height: 640px` on `/documents/[id]/review` (taller than Phase 2 detail view because the operator must read while editing). Value is a multiple of 8, not 16 — accepted as a readability constraint for side-by-side inspection.
- Submit button min-height: inherits shadcn default (40px).
- Field-edited indicator border: `2px` left-border accent on modified field (outside grid; component-native focus-ring-style affordance).

## Color

60/30/10 split unchanged. **No new color tokens introduced in Phase 3.** All warning/destructive/accent semantics are reused verbatim from Phase 2.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, main surface |
| Secondary (30%) | `#F5F5F5` (neutral-100) | Card backgrounds (form card, authority result card, PDF preview card surround), breadcrumb chip backgrounds |
| Accent (10%) | `oklch(0.21 0.04 265)` (slate-900) | Primary CTA `Speichern & Behörde ermitteln`; focus rings; left-border accent on modified form fields (2px) |
| Destructive | `oklch(0.58 0.22 27)` (red-600) | Inline Zod validation errors; not-found panel heading + icon; unexpected-error toast icon |
| Warning | `oklch(0.77 0.15 80)` (amber) | `[PRÜFEN]` banner fill; ambiguous-match panel banner fill; medium-confidence badges carried from Phase 2 extracted values |

### Accent — reserved for (unchanged + extended)

- Primary CTA button `Speichern & Behörde ermitteln`.
- Focus ring on all interactive elements (inputs, selects, buttons, links) — inherited from shadcn defaults.
- 2px left border on form fields whose value differs from the original Claude extraction (per CONTEXT D-10 "modified field gets a subtle accent border").
- Clickable `tel:`, `mailto:`, and website links in the authority contact block — underline on hover + accent color for link text.

**Accent is NOT used for:** secondary CTA `Verwerfen` (ghost/outline variant), card borders, body text, breadcrumb text, section headers, or status-panel icons (not-found / ambiguous).

### Destructive — reserved for (unchanged + extended)

- Inline Zod field-validation error copy below inputs.
- Not-found authority panel: heading `Keine Behörde gefunden`, icon (lucide `CircleAlert`).
- Unexpected-error Sonner toast icon.
- Missing-value placeholder `— nicht erkannt` carried from Phase 2, if a field still displays it post-extraction.

### Warning — reserved for (unchanged + extended)

- Medium-confidence badges on the extracted-values context (where shown alongside editable inputs).
- `[PRÜFEN]` banner on resolved authorities where `needs_review=true` (source was marked with `[PRÜFEN]` — per CONTEXT D-05). Banner copy: `Angaben bitte prüfen`.
- Ambiguous-match banner on the authority result panel when multiple candidates returned (`lookup_status='ambiguous'`).
- Special-rules callout Badge (e.g., Führungszeugnis exception) uses warning variant for the callout only.

**Warning is NOT used for:** section backgrounds, form borders, field-edited indicators (that role is accent), or the `not_found` panel (that role is destructive).

### Status/confidence badge palette (carried forward)

Confidence badges on the extracted-values read-out beside each editable input follow the Phase 2 contract exactly:

| Level | Label (DE) | Badge variant | Fill | Text |
|-------|------------|---------------|------|------|
| high | `Hoch` | `secondary` | `#F5F5F5` | `#0A0A0A` |
| medium | `Mittel` | `warning` | `oklch(0.77 0.15 80)` | `oklch(0.2 0 0)` |
| low | `Niedrig` | `destructive` | `oklch(0.58 0.22 27)` | `#FFFFFF` |

## Interaction Contract

| Interaction | Expected Behaviour |
|-------------|--------------------|
| Page load with `extraction_status=done` | Form pre-populated with all 6 extracted values; missing values render input with empty string and a subtle destructive helper `— nicht erkannt` above the input |
| Page load with `extraction_status!=done` | Redirect to `/documents/[id]` (review entry is only valid for done-status documents per CONTEXT D-07) |
| Field edit | On first keystroke/change that differs from original, field gains 2px accent left border + `Ursprünglich: {value}` helper caption |
| Revert an edited field to original | Accent left border removed; helper caption hidden |
| Select component focus | Inherits shadcn focus ring (accent) |
| Date input | `type="date"`; browser renders native picker; value stored as ISO `yyyy-MM-dd` and displayed in German `dd.MM.yyyy` on the authority result panel |
| Submit (primary CTA) | Button disables, inline spinner, text swaps to `Behörde wird ermittelt …`. Server Action validates with Zod, persists `document_review`, runs resolver, returns `{ok, data}` union |
| Submit success (matched) | Success toast `Zuständige Behörde ermittelt.`; authority result panel renders below form with smooth scroll to panel top; form remains visible and editable (operator may re-run lookup after further edits) |
| Submit success (ambiguous) | Same flow but panel variant is the ambiguous panel with candidate list |
| Submit success (not_found) | Warning toast not needed — panel alone conveys. Panel renders with destructive heading |
| Submit validation failure | No server call. Inline error text under first invalid field. Focus moves to first invalid field. Button re-enables |
| Submit server error | Destructive toast `Behörde konnte nicht ermittelt werden. Bitte erneut versuchen.`; button re-enables; form state preserved |
| Click `Verwerfen` with unsaved edits | Opens discard confirmation dialog. On confirm, all fields reset to original extraction values; border indicators clear; any authority result panel is dismissed |
| Click `Verwerfen` with no edits | No-op (button is disabled when no field differs from original) |
| Click phone link | `tel:{phone}` native handler |
| Click email link | `mailto:{email}` native handler |
| Click website link | Opens in new tab (`target="_blank"`, `rel="noopener noreferrer"`) |
| Click `Eingaben anpassen` on not-found panel | Smooth scroll to form; focus moves to `Dokumenttyp` select |
| Click `Diese Behörde übernehmen` on ambiguous panel | Persists the chosen authority to the existing `document_review` row; panel transitions to matched variant; success toast fires |
| Keyboard navigation — form | Tab order: Dokumenttyp → Ausstellende Behörde → Ausstellungsort → Bundesland → Ausstellungsdatum → Voller Name → Verwerfen → Speichern & Behörde ermitteln |
| Keyboard navigation — post-submit | Result panel receives focus on render (announced by screen reader as heading); Tab into candidate list or action buttons |
| Focus visible | Accent focus ring on all interactive elements (inherited from shadcn) |
| Browser refresh with unsaved edits | `beforeunload` warning `Änderungen gehen verloren. Seite verlassen?` if any field differs from original |

## Layout Contract

### Review page (`/documents/[id]/review`)

- Reuse Phase 1 header shell unchanged.
- Breadcrumb row (Body size, muted): `Übersicht / Dokument / Überprüfung`.
- Heading block: `Überprüfung` (24px/600) + filename + upload timestamp subtext (muted Body size).
- **Two-column grid on ≥1024px:**
  - Left column: PDF preview card (flex 1).
  - Right column: form card, `min-width: 420px`, `max-width: 520px`.
  - Gap between columns: `32px` (xl).
- **Below 1024px:** stack vertically — PDF preview first, form second. Gap `24px` (lg).
- Left column — PDF preview card: shadcn Card with `0` internal padding; iframe fills; min-height `640px`.
- Right column — form card: shadcn Card with padding `24px` (lg); internal form grid with `16px` (md) row gap. Section label `Extrahierte Daten` (24px/600) at top of card. CTAs row at bottom of card: `Verwerfen` (ghost variant, left-aligned) and `Speichern & Behörde ermitteln` (default accent variant, right-aligned). `16px` gap between buttons. Full-width on <640px stacked.
- **Authority result panel:** rendered below the two-column grid in a full-width row. Gap from grid: `32px` (xl). Card with padding `24px` (lg).
  - Matched variant internal layout:
    - Routing breadcrumb strip (Label size, muted) — at top of card.
    - Authority name (Heading 24px/600) — directly below.
    - 2-column definition list on ≥640px (label column fixed `128px`, value column flex); stacked on <640px. Row gap `8px` (sm).
    - Special rules callout (if present): full-width warning Badge + Body copy, with `16px` (md) top margin.
    - `[PRÜFEN]` banner (if `needs_review`): full-width warning-fill strip with label `Angaben bitte prüfen` + subtext; positioned at top of card above breadcrumb.
  - Not-found variant: centered column, icon (lucide `CircleAlert`, 24px, destructive) + heading + body + `Eingaben anpassen` outline button. Vertical padding `32px` (xl).
  - Ambiguous variant: warning banner at top + candidate list using shadcn Table with columns `Behörde | Routing | Aktion` where Aktion column renders `Diese Behörde übernehmen` button (sm variant).

### Component sizing (inherited)

- Button default: 40px height.
- Button sm: 36px height.
- Input default: 40px height.
- Select trigger: 40px height (inherits Input sizing).
- Badge: 20px height.
- Table row: 44px height (carried from Phase 2).

## Field-Edited Indicator Specification

Precise because this is the only net-new visual contract introduced in Phase 3.

| State | Visual |
|-------|--------|
| Unchanged | Default shadcn Input/Select — no left border accent; helper caption hidden |
| Edited | 2px solid accent (`oklch(0.21 0.04 265)`) on the left edge of the input container; `Ursprünglich: {original_value}` caption renders below the input in muted Body size; caption color `#525252` |
| Edited + validation error | Left-border accent retained; destructive inline error takes precedence visually below the input; `Ursprünglich:` caption moves below the error (stacked) |
| Edited + focused | Left-border accent remains; shadcn focus ring also applies (ring does not conflict — ring wraps all 4 sides, left border is inside) |

The indicator is informational (operator can see at a glance which fields they changed). It is not an error signal.
