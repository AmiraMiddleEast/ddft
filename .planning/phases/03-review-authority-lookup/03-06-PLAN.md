**Walkthrough checklist (11 items):**

1. **Navigation to review page**
   - Go to `/documents/[id]` for a done-status document.
   - Confirm the "Zur Überprüfung" button is ENABLED (no longer greyed out).
   - Click it. You should land at `/documents/[id]/review`.
   - Expected URL tab title: "Überprüfung — Angela".

2. **Two-column layout (≥1024px viewport)**
   - Left column shows a PDF iframe of the original document (min height ~640px).
   - Right column shows the form "Extrahierte Daten" with 6 editable fields.
   - Column gap looks like about 32px; both columns align at the same top.

3. **Stacked layout (<1024px viewport)**
   - Resize the window to ~900px wide.
   - PDF now appears ABOVE the form; both take full width.

4. **Field editing + accent-border indicator**
   - Click into the Dokumenttyp Select. The list is alphabetical and contains real document types (Geburtsurkunde, Approbationsurkunde, etc.).
   - Change the selection. The field's container gains a 2px left border in the accent color.
   - A muted caption appears below: "Ursprünglich: {the prior Claude-extracted value}".
   - Revert to the original value — the border + caption disappear.

5. **Bundesland Select**
   - Contains all 16 German states alphabetically, PLUS a sentinel "Unbekannt / Sonstiges" option at the end (or somewhere visible).

6. **Confidence badges**
   - Beside each field label, a badge shows the Phase 2 confidence level: Hoch / Mittel / Niedrig.

7. **Submit → matched result**
   - Set sensible values (e.g. Dokumenttyp=Approbationsurkunde, Bundesland=Bayern, Ausstellungsort=München).
   - Click "Speichern & Behörde ermitteln".
   - Button disables with spinner + text "Behörde wird ermittelt …".
   - Success toast appears: "Zuständige Behörde ermittelt."
   - AuthorityResultPanel renders below the form with:
     - Heading "Zuständige Behörde ermittelt"
     - Routing breadcrumb using ›: e.g. "Bayern › Oberbayern › Approbationsurkunde"
     - Authority name in large heading
     - Anschrift, Telefon (tel: link if present), E-Mail (mailto: link if present), Website (new-tab link if present), Öffnungszeiten in a 2-column definition list
     - If present: special-rules callout in warning color
     - If present: PRÜFEN banner "Angaben bitte prüfen" in amber

8. **Submit → ambiguous result**
   - Change Ausstellungsort to an unknown small city (e.g. "Kleinsdorf") but keep state = Bayern.
   - Submit. Result panel switches to "Mehrere Behörden möglich" with a warning banner and a candidate list table.
   - Each row has a "Diese Behörde übernehmen" button.
   - Click one. Panel transitions to the matched variant; success toast fires.

9. **Submit → not_found result**
   - Change Bundesland to "Unbekannt / Sonstiges".
   - Submit. Result panel shows "Keine Behörde gefunden" with destructive icon + CTA "Eingaben anpassen".
   - Click the CTA; page scrolls to the form; focus lands on the Dokumenttyp field.

10. **Verwerfen + DiscardDialog**
    - Edit a field so "Verwerfen" is enabled.
    - Click "Verwerfen". A dialog appears with "Änderungen verwerfen?" + body + "Verwerfen" / "Abbrechen" buttons.
    - Click "Abbrechen" — dialog closes, edits preserved.
    - Click "Verwerfen" again — dialog closes, ALL edits reverted, any result panel dismissed.

11. **beforeunload warning**
    - Edit a field. Attempt to close the tab or navigate away.
    - Browser prompts "Änderungen gehen verloren. Seite verlassen?" (browser-specific wording; core behavior is that the prompt fires).
