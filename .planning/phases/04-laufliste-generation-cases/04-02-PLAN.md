<task type="auto" tdd="true">
  <name>Task 2: Server Actions — create, add, remove, reorder</name>
  <files>lib/cases/actions.ts, lib/cases/actions.test.ts</files>
  <behavior>
    - createCaseAction({ personName, personBirthdate?, notes? }) validates via Zod (personName min 1 max 200, personBirthdate optional ISO yyyy-MM-dd, notes max 2000), inserts a case row with status='open', returns { ok: true, caseId }.
    - Empty person_name returns { ok: false, error: 'VALIDATION' }.
    - No session returns { ok: false, error: 'UNAUTHORIZED' }.
    - addDocumentsToCaseAction({ caseId, documentIds: string[] }) verifies case ownership, validates each doc is owned + approved + unassigned, then inserts case_document rows with positions = maxExistingPosition+1, maxExistingPosition+2, ... in a single transaction.
    - If any documentId is already in another case_document (UNIQUE constraint SQLITE_CONSTRAINT_UNIQUE), the transaction rolls back and returns { ok: false, error: 'DOC_ALREADY_ASSIGNED', details: { documentId } }.
    - Mixed ownership: documentIds not owned by caller → { ok: false, error: 'FORBIDDEN' }, transaction rolled back.
    - removeDocumentFromCaseAction({ caseId, caseDocumentId }) verifies case ownership, deletes the row in a transaction, then re-numbers remaining rows contiguously (1..N by existing position order).
    - reorderCaseDocumentsAction({ caseId, caseDocumentId, direction: 'up' | 'down' }) finds the adjacent row and swaps positions atomically inside db.transaction. No-op at edges (first row 'up' / last row 'down') returns { ok: true, noop: true }.
    - Reorder uses the three-step dance (temp pos=-1) to avoid any transient duplicate if a unique (case_id, position) partial index gets added later (Pitfall 4).
  </behavior>
