# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Laufliste Generation & Cases.
> Web surfaces: `/cases`, `/cases/new`, `/cases/[id]`. PDF layout contract: Laufliste document generated by `@react-pdf/renderer` (A4 portrait, Helvetica, 20mm margins).
> Extends Phases 1–3 for all web UI tokens (spacing, typography, color, focus ring). No new color tokens in the web UI. The PDF contract declares its own independent type scale because @react-pdf/renderer does not share Tailwind tokens.
> Generated by gsd-ui-researcher (autonomous mode).

---

## Surfaces in Scope

| Surface | Route / Context | Purpose |
|---------|-----------------|---------|
| Cases list | `/cases` | Table of all cases owned by the user; entry points to create/open |
| Create case | `/cases/new` | Minimal form: person name (required), birthdate (optional), notes (optional) |
| Case detail | `/cases/[id]` | Person header + documents list (add/remove/reorder) + "Laufliste generieren" CTA + Historie of generated Lauflisten |
| Add-documents picker | `/cases/[id]` (sheet/dialog) | Multi-select picker of reviewed, unassigned documents |
| Inline validation errors | `/cases/new`, `/cases/[id]` | Zod validation failures under fields |
| Toasts | Global (Sonner) | Case created, document added/removed, Laufliste generated, errors |
| **Laufliste PDF** | Output artifact (not a route) | A4 German routing slip generated by `@react-pdf/renderer`, downloaded via `/api/cases/[id]/laufliste/[lauflisteId]/download` |

Out of scope (deferred): bilingual PDF, in-browser PDF preview before download, email-to-customer, digital signature, Behörden admin CRUD.

## Color (Web UI)

60/30/10 split unchanged. **No new color tokens introduced in Phase 4.** Accent, destructive, and warning semantics are inherited verbatim.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, main surface |
| Secondary (30%) | `#F5F5F5` (neutral-100) | Card backgrounds, table header row, case-header info card |
| Accent (10%) | `oklch(0.21 0.04 265)` (slate-900) | Primary CTAs (`Fall anlegen`, `Laufliste generieren`, `Herunterladen`, `Erneut generieren`); focus rings |
| Destructive | `oklch(0.58 0.22 27)` (red-600) | `Entfernen` destructive-variant button text; destructive confirm button in remove-document dialog; validation errors; "konnte nicht erzeugt werden" error toast icon |
| Warning | `oklch(0.77 0.15 80)` (amber) | Status badge for `open` case state (`In Bearbeitung`); inline banner if a case has a document not yet approved (blocks Laufliste generation) |

### Accent — reserved for (unchanged + extended)

- Primary CTAs: `Fall anlegen` (on `/cases/new`), `Laufliste generieren` (on `/cases/[id]`), `Herunterladen` (last-generated), `Erneut generieren`, `Dokumente hinzufügen`.
- Focus ring on all interactive elements (inherited from shadcn defaults).
- Selected row highlight in the add-documents picker (2px accent left border on focused row; `bg-accent/10` on checked row).

**Accent is NOT used for:** secondary CTAs (`Abbrechen`, `Zurück`, the `Öffnen` row action), the reorder arrows, card borders, body text, section headers, or the case-status badges.

### Destructive — reserved for (unchanged + extended)

- `Entfernen` row action on each document within a case (ghost button with destructive text color).
- Confirmation dialog confirm button for removing a document.
- Confirmation dialog confirm button for regenerating Laufliste when a prior one exists (non-destructive but emphasized so operator is aware a new file is created).
- Inline Zod validation errors.
- Destructive Sonner toast icon for unexpected server errors (e.g., PDF render failure).

### Warning — reserved for (unchanged + extended)

- Status badge `In Bearbeitung` on the cases list and on `/cases/[id]` header for cases with status `open`.
- Inline blocker banner on `/cases/[id]` when the case contains a document whose `review_status != 'approved'`. Banner copy: `Mindestens ein Dokument ist noch nicht geprüft.`
- Inline blocker banner on `/cases/[id]` when the case has zero documents. Banner copy: `Bitte mindestens ein Dokument hinzufügen.`

**Warning is NOT used for:** section backgrounds, card borders, the `Historie` state, or the document-row default background.

### Status badge palette (neutral semantics for case status)

| `case.status` | Label (DE) | Badge variant | Rationale |
|---------------|------------|---------------|-----------|
| `open` | `In Bearbeitung` | `warning` (amber fill) | Review-worthy attention — not shouting red |
| `ready_for_pdf` | `Bereit` | `outline` | Ready but non-shouting |
| `pdf_generated` | `Laufliste erstellt` | `secondary` | Calm completion state |

## Interaction Contract (Web UI)

| Interaction | Expected Behaviour |
|-------------|--------------------|
| `/cases` page load | Server-render table sorted `updated_at DESC`. Empty-state card renders when no cases exist. |
| Click `Neuen Fall anlegen` | Navigates to `/cases/new` |
| Submit create-case form | Server Action validates with Zod; creates `case` row scoped to `session.user.id`; redirects to `/cases/[id]`; success toast `Fall angelegt.` |
| Submit validation failure | No server call; inline error under first invalid field; focus moves there |
| `/cases/[id]` page load | Ownership check in server component; redirect to `/cases` if not owner. Render person header, documents list (ordered by `case_document.position ASC`), Laufliste section (latest first), Historie (older ones). |
| `/cases/[id]` load — blocker state | If document count = 0 → show empty-documents card + warning banner; `Laufliste generieren` CTA disabled. If any document `review_status != 'approved'` → show warning banner + CTA disabled with tooltip `Alle Dokumente müssen geprüft sein.` |
| Click `Dokumente hinzufügen` | Opens sheet; fetches reviewed + unassigned documents for this user; renders table with checkboxes. |
| Select documents in picker | Each click toggles selection; primary CTA label updates to `{n} hinzufügen`; Enter on a focused checkbox toggles it. |
| Submit picker | Server Action transactionally inserts `case_document` rows with incremented positions; if any selected doc was just assigned to another case (race), returns per-row error and surfaces toast `Dokument ist bereits einem anderen Fall zugeordnet.` and re-renders list. |
| Click up-arrow on a row | Swaps position with row above; optimistic UI reorder; Server Action persists both `position` integers in a transaction. No arrow on first row (rendered disabled). |
| Click down-arrow on a row | Symmetric. No arrow on last row. |
| Keyboard on arrow buttons | Enter/Space triggers move; focus retained on the button (which now represents the row's new position); screen reader announces new position via live region `Position {n} von {total}.` |
| Click `Entfernen` on a row | Opens remove-document dialog; on confirm, Server Action deletes `case_document` row; remaining rows renumbered contiguously; success toast `Dokument entfernt.` |
| Click row or `Ansehen` | Navigates to `/documents/[id]` |
| Click `Laufliste generieren` (first time) | Button disables + inline spinner + text swaps to `Laufliste wird erstellt …`. Server Action: re-runs authority resolver for each document (CONTEXT D-18), renders PDF via `@react-pdf/renderer` → `renderToBuffer`, writes file to `data/lauflisten/{caseId}-{timestamp}.pdf`, inserts `laufliste` row, updates `case.status='pdf_generated'`. On success: success toast; Laufliste section re-renders with `Herunterladen` + `Erneut generieren`; Historie gains new row. |
| Click `Erneut generieren` | Opens regenerate dialog; on confirm, same flow; prior file remains on disk and in Historie. |
| Click `Herunterladen` | Browser navigates to `/api/cases/[id]/laufliste/[lauflisteId]/download`. Route Handler streams with `Content-Disposition: attachment; filename*=UTF-8''Laufliste-{person-slug}-{date}.pdf`. On HTTP error: destructive toast `Download fehlgeschlagen. Bitte erneut versuchen.` |
| Server error during generate | Button re-enables; destructive toast `Laufliste konnte nicht erzeugt werden. Bitte erneut versuchen.`; no `laufliste` row written (atomic) |
| Browser refresh mid-generate | Action is a single Server Action request — either completed on server (refresh shows new Laufliste) or not (no row). No intermediate state to recover. |
| Keyboard navigation — case detail | Tab order: breadcrumb `Fälle` link → person meta edit (out of scope — no edit in v1) → `Dokumente hinzufügen` → per row: open link → up arrow → down arrow → remove → `Laufliste generieren` → Historie rows (each `Herunterladen`) |
| Focus visible | Accent focus ring on all interactive elements (inherited from shadcn) |

## PDF Layout Contract (Laufliste output)

Generated by `@react-pdf/renderer` on the server. The PDF declares its own isolated type scale and spacing because it cannot share Tailwind tokens.

### Document properties

| Property | Value |
|----------|-------|
| Page size | A4 (210 × 297 mm) |
| Orientation | Portrait |
| Margins | `20mm` on all four sides |
| Font family | `Helvetica` (built-in) — supports ä, ö, ü, ß via default WinAnsi encoding |
| Page number footer | `Seite {pageNumber} von {totalPages}` — centered, Caption size, muted |
| Language | German (all strings; CONTEXT D-19) |

### PDF type scale (independent of web tokens)

| Role | Size | Weight | Line height | Usage |
|------|------|--------|-------------|-------|
| Main heading | 18pt | bold | 1.2 | Page-1 title `Laufliste` |
| Section title | 12pt | bold | 1.3 | Per-document section header `{n}. Dokument — {Dokumenttyp}`; step labels `Vorbeglaubigung`, `Endbeglaubigung`, `Legalisation durch VAE-Botschaft` |
| Body | 10pt | regular | 1.4 | Address lines, phone, email, hours, special-rules text, person name subheading |
| Emphasis (inline) | 10pt | bold | 1.4 | Key/value labels (`Anschrift`, `Telefon`, `E-Mail`, `Öffnungszeiten`, `Besondere Hinweise`, `Dokumentart`, `Person`, `Erstellt am`); `[PRÜFEN]` marker |
| Caption | 8pt | regular | 1.3 | Footer page number |

Exactly 4 sizes (18/12/10/8) and exactly 2 weights (regular/bold) — matches the web UI discipline even though the scales differ.

### PDF color

Monochrome with one muted accent per CONTEXT D-Discretion and the task spec.

| Role | Value | Usage |
|------|-------|-------|
| Text primary | `#111111` | All body and headings |
| Text muted | `#555555` | Footer, captions, `— nicht hinterlegt` empty values |
| Muted accent | `#4B5563` (cool gray 600) | Section title underlines + step label underlines only — not used as fill |
| Divider / rule | `#BFBFBF` | Horizontal hairline separators between sections and steps (0.75pt) |
| Warning marker (`[PRÜFEN]`) | `#111111` on `#FDE68A` (amber-200) fill | Inline pill `[PRÜFEN]` before authority name in Vorbeglaubigung block only when `needs_review=true` |

No additional color. No logo. No images.

### Page 1 header (appears once on page 1 only)

- **Row 1:** `Laufliste` (Main heading 18pt bold), left-aligned.
- **Row 2 (gap 8pt):** `Person: {person_name}` (Body 10pt with bold label).
- **Row 3:** `Geburtsdatum: {dd.MM.yyyy}` — omitted if not stored.
- **Row 4:** `Erstellt am: {dd.MM.yyyy}` (Body 10pt with bold label).
- **Row 5:** `Dokumente insgesamt: {n}` (Body 10pt with bold label).
- **Divider:** full-width 0.75pt rule beneath, `16pt` space below.

### Footer (every page)

- Centered `Seite {pageNumber} von {totalPages}` — Caption 8pt muted.
- Fixed position bottom of page within 20mm bottom margin.

### Per-document section (repeats for each document in `case_document.position` order)

Each section is kept together on a single page where possible (`wrap={false}` when height allows); if too tall, it can break between steps.

1. **Section title bar:** `{n}. Dokument — {Dokumenttyp}` (Section 12pt bold) with 0.5pt muted-accent underline.
2. **Document metadata block** (Body 10pt, key/value with 2pt row gap):
   - `Dokumentart: {Dokumenttyp}`
   - `Ausstellende Behörde: {ausstellende_behoerde}`
   - `Ausstellungsort: {ausstellungsort}`
   - `Ausstellungsdatum: {dd.MM.yyyy}`
   - `Voller Name: {voller_name}`
   - Missing values render as `— nicht erkannt` in muted color.
3. **Step 1 — Vorbeglaubigung** (Section 12pt bold `Vorbeglaubigung`, muted-accent underline):
   - `[PRÜFEN]` pill (amber fill) inline before authority name **only if** `needs_review=true`.
   - Authority name (Emphasis 10pt bold).
   - `Anschrift`: multi-line street + PLZ + city.
   - `Telefon`, `E-Mail`, `Website`, `Öffnungszeiten` — each on its own line; empty fields render `— nicht hinterlegt` in muted.
   - `Besondere Hinweise`: free-text block (Body 10pt), only when present.
4. **Step 2 — Endbeglaubigung** (Section 12pt bold, muted-accent underline):
   - Static block: Bundesverwaltungsamt (Köln) for most documents; Bundesamt für Justiz (Bonn) for Führungszeugnis (per CONTEXT D-06).
   - Same key/value fields as step 1. Hardcoded from `lib/laufliste/endbeglaubigung.ts`.
5. **Step 3 — Legalisation durch VAE-Botschaft** (Section 12pt bold, muted-accent underline):
   - Static block: Botschaft der Vereinigten Arabischen Emirate, Berlin. Hardcoded from `lib/laufliste/embassy.ts`.
6. **Section trailing divider:** 0.75pt muted rule, `16pt` space above the next section.

### File-name contract

`Laufliste-{person-slug}-{yyyy-MM-dd}.pdf`

- `person-slug`: lowercase ASCII transliteration of `person_name` (ß→ss, ä→ae, ö→oe, ü→ue), spaces→`-`, strip punctuation.
- Served via RFC 5987 `Content-Disposition: attachment; filename*=UTF-8''…` (per CONTEXT D-15) so non-ASCII originals are preserved as a fallback.
