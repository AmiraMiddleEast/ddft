# Plan 05-01 — Schema + nuqs install

<objective>
Extend the Drizzle schema with `document_version` table and add `version` column to `document`. Install nuqs for URL state management. Apply migration. This is foundational for Phase 5's three sub-features (history, re-upload, admin).
</objective>

<must_haves>
- `document_version` table exists with: id TEXT PK, documentId FK CASCADE DELETE, versionNumber INT, storagePath TEXT, sha256 TEXT, size INT, uploadedAt INTEGER. UNIQUE index on (documentId, versionNumber).
- `document` table has `version INTEGER DEFAULT 1 NOT NULL` column.
- `nuqs` is installed and pinned in package.json.
- Migration applied successfully — `sqlite3 data/angela.db ".tables"` shows `document_version`.
- `npx tsc --noEmit` clean.
- Full vitest suite passes (`BETTER_AUTH_SECRET=testsecret12345678901234567890 npx vitest run`).
</must_haves>

<task id="05-01-01" type="auto">
<action>
1. Read the current `db/schema.ts` file.
2. Append a new `sqliteTable` named `documentVersion` (table name `"document_version"`) with columns per the must-haves, using the existing pattern (id via `text("id").primaryKey()`, FK references `document.id` with `onDelete: "cascade"`, timestamps via `integer("uploaded_at").notNull()`).
3. Add a `uniqueIndex("doc_version_doc_num_uniq")` on `(documentId, versionNumber)`.
4. Add `version: integer("version").notNull().default(1)` to the existing `document` table.
5. Add a Drizzle `relations` block for documentVersion back to document.
6. Save the file.
7. Run `npx drizzle-kit generate` — a new migration SQL should appear under `drizzle/`.
8. Run `npx drizzle-kit push --force` to apply to dev DB. If push errors about state mismatch, delete `data/angela.db` (gitignored) and re-run push.
9. Verify with `sqlite3 data/angela.db ".tables"` — should list document_version.
10. Commit atomically: `feat(05-01): add document_version schema + document.version`.
</action>
