# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation & Authentication.
> Surfaces in scope: `/login` page, authenticated layout shell at `/`, inline and toast error states.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Surfaces in Scope

| Surface | Route | Purpose |
|---------|-------|---------|
| Login screen | `/login` | Unauthenticated operator signs in with email + password |
| Application shell | `/` (and all future authenticated routes) | Authenticated layout with header (user email + logout) and empty body placeholder ("Willkommen") |
| Inline login error | `/login` (in-form) | "E-Mail oder Passwort ungültig." rendered under inputs on 401 |
| Toast (unexpected errors) | Global | Sonner toast for network/server errors |
| Loading state | `/login` submit button | Disabled button with inline spinner during auth request |

Out of scope for Phase 1 (deferred to later phases): upload surfaces, review surfaces, PDF preview, admin surfaces.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon–label gap, tight inline padding |
| sm | 8px | Input-to-label gap, compact vertical rhythm |
| md | 16px | Default form field gap, card inner padding-block |
| lg | 24px | Card inner padding, header height padding-block |
| xl | 32px | Login card padding, layout top offset |
| 2xl | 48px | Vertical gap between card and page edges on large viewports |
| 3xl | 64px | Not used in Phase 1 (reserved for later phases) |

Exceptions: Login button minimum height 40px to satisfy 44px effective touch target with border (button text line-height + vertical padding resolves to ≥40px block).

## Typography

Exactly 3 sizes, exactly 2 weights. Line-height 1.5 for body/label, 1.2 for heading.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Input text, helper copy, toast message, "Willkommen" placeholder |
| Label | 14px | 600 (semibold) | 1.5 | Form labels ("E-Mail", "Passwort"), header user email, button label |
| Heading | 24px | 600 (semibold) | 1.2 | Login card title ("Anmelden"), future page titles |

No display tier in Phase 1 (no marketing surfaces).

## Color

60/30/10 split. Neutral professional palette — internal tool, not marketing. Values use OKLCH via Tailwind v4 `@theme`; hex equivalents listed for reference and checker validation.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (light) / `#0A0A0A` (dark mode deferred to post-v1) | Page background, main surface |
| Secondary (30%) | `#F5F5F5` (neutral-100) | Login card background, header background, input background |
| Accent (10%) | `#0F172A` (slate-900) | Primary button fill, focus ring |
| Destructive | `#DC2626` (red-600) | Inline login error text, destructive toast icon |

**Accent reserved for:** primary button ("Anmelden"), focused input border/ring, focused link. **Not** used for body text, headers, borders, or decorative elements.

**Text-on-color:**
- Text on dominant: `#0A0A0A` (near-black)
- Text on accent: `#FFFFFF`
- Muted text (e.g., helper copy): `#525252` (neutral-600)
- Border default: `#E5E5E5` (neutral-200)

Light mode only in v1. Dark mode deferred.

## Interaction Contract

| Interaction | Expected Behaviour |
|-------------|--------------------|
| Email input focus | Accent focus ring, no layout shift |
| Password input type | `type="password"`, no visibility toggle in v1 |
| Submit (Enter key) | Form submits; same path as clicking the button |
| Submit in-flight | Button disabled, shows inline spinner, text becomes `Anmelden…`. Inputs remain enabled (can be edited on error return). |
| Invalid credentials response | Inline error renders below password field, inputs retain values, focus returns to email input |
| Rate-limited response | Inline rate-limit copy replaces inline error; button re-enables after 60s |
| Successful login | Client navigates to `/` — no intermediate success toast |
| Logout click | POST to `/api/auth/sign-out`, redirect to `/login`, show `Sie wurden abgemeldet.` toast |
| Unauthenticated access to any non-`/login` route | Middleware redirect to `/login` — no flash of protected content |
| Keyboard navigation | Tab order: email → password → submit → logout (on `/`). All interactive elements reachable without mouse. |
| Focus visible | Accent focus ring on all interactive elements (inherited from shadcn defaults — do not disable) |

## Layout Contract

**Login page (`/login`):**
- Full viewport, vertically and horizontally centered card
- Card max-width: `360px`
- Card padding: `32px` (xl)
- Card gap between elements: `16px` (md)
- Page background: dominant; card background: secondary
- No header, no footer, no navigation on `/login`

**Application shell (`/` and future authenticated routes):**
- Header: full-width, `64px` height, horizontal padding `24px` (lg), secondary background, border-bottom `1px solid` border default
- Header left: app name `Angela` in label style (14px/600) — no logo in v1
- Header right: user email (label style) followed by `Abmelden` button (ghost variant)
- Main content area: full-width below header, default page padding `24px` (lg)
- Home body: centered `Willkommen` heading + muted subtext
