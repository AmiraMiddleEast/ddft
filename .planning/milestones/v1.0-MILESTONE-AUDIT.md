## Known Technical Debt (acknowledged, non-blocking)

1. **Synthetic Behörden data** — `data/behoerden-parsed.json` was generated with a
   placeholder ANTHROPIC_API_KEY. Production requires running
   `tsx scripts/seed-behoerden.ts --force` with a real key so authority lookups
   return correct contact information for the 16 Bundesländer.

2. **BVA/BfJ/UAE Embassy contact defaults** — Phase 4 hardcoded research defaults
   with `@assumed` JSDoc for the Endbeglaubigung and Legalisation steps. Operator
   must verify against an actual sample Laufliste PDF before the first production
   print run.

3. **Path-containment defense-in-depth gap** — R-04-01 and R-05-01 identify a
   warning-level gap in two read-side PDF routes. Not exploitable today because
   every writer is server-generated (UUID-based). One `startsWith` assertion on
   each route would close the gap; tracked for a cross-cutting security hardening
   pass.

4. **38 UAT items pending operator sign-off** — auto-approved in autonomous mode
   per operator instruction; each phase has a `HUMAN-UAT.md` file listing the
   items to walk through in a real-browser session.

findings across all 5 phases. The single recurring warning (path containment
on server-generated-only file paths) is documented as acknowledged debt.
